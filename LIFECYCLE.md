# CodeNomad Tunnel - 完整生命周期

## 一、两个服务

```
┌─────────────────────────────────────────────────────────────────────┐
│                        两个服务各司其职                               │
├─────────────────────────────────┬───────────────────────────────────┤
│   服务 A: tunnel-hub (远端)      │   服务 B: CodeNomad (本地)         │
│   位置: 公网 VPS                │   位置: 用户电脑/内网服务器         │
│   职责: 接收公网流量 + 转发       │   职责: 连接隧道 + 转发到本地服务   │
│   语言: Go 单二进制              │   职责: 管理隧道生命周期           │
│   端口: :80(公网) :8080(API)     │   职责: UI 展示 + 交互             │
│   启动: ./tunnel-hub \          │   端口: :9899                      │
│     --domain tunnel.xx.com      │   启动: npx codenomad --launch ... │
└─────────────────────────────────┴───────────────────────────────────┘

不需要 Docker！直接二进制启动：

  # 远端 VPS 上
  ./tunnel-hub --domain tunnel.yourdomain.com --http-port 80 --api-port 8080

  # 本地
  npx @dyyz1993/codenomad@0.15.34 --launch --password 123456 --http true ...
```

## 二、服务启动生命周期

### tunnel-hub 启动 (远端 VPS)

```
./tunnel-hub --domain tunnel.yourdomain.com
         │
         ▼
    ┌─────────────┐
    │ 解析 CLI 参数 │  --domain, --http-port, --api-port, --tls-cert, --tls-key
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ 创建 Hub 实例 │  内存 Map, 清理定时器, 统计计数器
    └──────┬──────┘
           │
           ▼
    ┌──────────────────────────────────────┐
    │ 启动 3 个监听器                       │
    │                                      │
    │  :80   → HTTP Proxy Handler          │
    │          接收 *.tunnel.xx.com 请求    │
    │          提取子域名 → 查找隧道 → 转发  │
    │                                      │
    │  :8080 → Management API              │
    │          POST/GET/DELETE /api/tunnels │
    │          GET /api/tunnels/:id/logs    │
    │          GET /api/health              │
    │                                      │
    │  /relay/:id → WebSocket Handler       │
    │          等待本地客户端连接            │
    └──────┬───────────────────────────────┘
           │
           ▼
    ┌──────────────────────────────────────┐
    │ 启动后台清理协程 (每 5 分钟)           │
    │                                      │
    │  遍历所有隧道:                        │
    │  - WebSocket 断开 > 5min → 删除隧道   │
    │  - 回收子域名                         │
    └──────┬───────────────────────────────┘
           │
           ▼
    ┌──────────────┐
    │ 就绪，等待请求 │
    └──────────────┘
```

### CodeNomad 启动 (本地，包含 tunnel-client 模块)

```
npx codenomad --launch --password 123456 --http true \
  --tunnel-hub-url https://tunnel.yourdomain.com
         │
         ▼
    ┌──────────────┐
    │ 正常启动流程   │  HTTP server, auth, workspaces...
    └──────┬───────┘
           │
           ▼
    ┌───────────────────────┐
    │ 初始化 TunnelClient    │
    │                       │
    │  读取配置:             │
    │  - tunnelHubUrl       │
    │  - enabled/disabled   │
    │                       │
    │  连接测试:             │
    │  GET {hubUrl}/api/health │
    │  ├─ 200 → 标记可用     │
    │  └─ 失败 → 标记不可用,  │
    │           不影响主服务   │
    └──────┬────────────────┘
           │
           ▼
    ┌──────────────┐
    │ 就绪，待命    │  隧道功能按需激活，不主动创建
    └──────────────┘
```

## 四、数据帧协议 (WebSocket Relay)

```
请求方向: tunnel-hub → CodeNomad client

  ┌─────────────────────────────────────────────────┐
  │ {                                               │
  │   "type": "request",                            │
  │   "id": "r_abc123",          ← 请求唯一标识      │
  │   "method": "GET",                              │
  │   "path": "/captcha/image",                     │
  │   "query": "t=123",                             │
  │   "headers": {                                  │
  │     "Accept": "text/html",                      │
  │     "Cookie": "session=abc"                     │
  │   },                                            │
  │   "body": ""                                    │
  │ }                                               │
  └─────────────────────────────────────────────────┘

响应方向: CodeNomad client → tunnel-hub

  文本响应:
  ┌─────────────────────────────────────────────────┐
  │ {                                               │
  │   "type": "response",                           │
  │   "id": "r_abc123",          ← 匹配请求 ID      │
  │   "status": 200,                                │
  │   "headers": {                                  │
  │     "Content-Type": "text/html"                 │
  │   },                                            │
  │   "body": "<html>...</html>"                    │
  │ }                                               │
  └─────────────────────────────────────────────────┘

  二进制响应 (图片等):
  ┌─────────────────────────────────────────────────┐
  │ {                                               │
  │   "type": "response",                           │
  │   "id": "r_abc123",                             │
  │   "status": 200,                                │
  │   "headers": {                                  │
  │     "Content-Type": "image/png"                 │
  │   },                                            │
  │   "bodyBase64": "iVBORw0KGgo..."  ← base64 编码 │
  │ }                                               │
  └─────────────────────────────────────────────────┘

错误/超时:
  ┌─────────────────────────────────────────────────┐
  │ tunnel-hub 等 30s 无响应 → 返回给公网客户端:     │
  │ HTTP 504 Gateway Timeout                        │
  │                                                 │
  │ relay 连接断开时:                                │
  │ HTTP 502 Bad Gateway                            │
  └─────────────────────────────────────────────────┘
```

## 六、直接启动 (不用 Docker)

```bash
# 编译
cd codenomad-tunnel
go build -o tunnel-hub .

# 直接运行
./tunnel-hub --domain tunnel.yourdomain.com

# 后台运行
nohup ./tunnel-hub --domain tunnel.yourdomain.com > tunnel.log 2>&1 &

# systemd 服务 (推荐)
sudo cat > /etc/systemd/system/tunnel-hub.service << 'EOF'
[Unit]
Description=CodeNomad Tunnel Hub
After=network.target

[Service]
Type=simple
ExecStart=/opt/tunnel-hub/tunnel-hub --domain tunnel.yourdomain.com
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable tunnel-hub
sudo systemctl start tunnel-hub
```
